RPpilot — готовый вариант с онлайн-регистрацией

Что изменено:
- больше нет фиксированного Static ID владельца;
- самый первый аккаунт в пустой базе автоматически становится главным;
- главный аккаунт получает роли owner, developer, admin, earnings, user;
- добавлена панель разработчика;
- главный аккаунт может выдавать «Панель разработчика» кому угодно;
- локальные аккаунты из старого сайта автоматически переносятся в базу при первом открытии после подключения;
- вход поддерживает старые локальные пароли и новые онлайн-пароли.

Файлы, которые должны быть на GitHub рядом с index.html:
- app.js
- styles.css
- supabase-config.js
- online-auth.js

В index.html уже должно быть:

<script src="app.js"></script>
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
<script src="supabase-config.js"></script>
<script src="online-auth.js"></script>

Как запустить базу:
1. Создай проект Supabase.
2. Открой SQL Editor -> New query.
3. Вставь код из supabase-schema.sql.
4. Нажми Run.
5. В supabase-config.js вставь Project URL и anon public key.

Важно:
Фиксированный главный Static ID больше не нужен.
Главным станет первый аккаунт, который появится в пустой таблице rppilot_users.
Если на устройстве уже есть локальный аккаунт старого сайта, он автоматически перенесётся в базу. Если база пустая, первый перенесённый локальный аккаунт станет главным.

Полностью без URL/key базы сайт не сможет сохранять аккаунты для всех пользователей. GitHub Pages не имеет своей базы данных и не умеет сам записывать пользователей в репозиторий.
